# plan.md — этапы разработки UNLHA

Каждый этап несёт законченную функциональность и проверяем в эмуляторе MAME
(через `make img`) или на реальном Sprinter. Этапы идут по возрастанию
сложности; SRAM-кэш подключается, когда базовая распаковка уже корректна.

Легенда статуса: [ ] не начато · [~] в работе · [x] готово.

---

## Этап 0 — Каркас сборки и «Hello, DSS»  [x]
**Цель:** рабочий пайплайн сборки и запуска.
- [x] Структура каталогов: `src/`, `include/`, `tools/`, `build/`, `test/`.
- [x] `Makefile` + `tools/build.sh`: `make exe` (sjasmplus `--raw` → `build/UNLHA.EXE`),
      `make img` (mtools → `build/unlha.img` с EXE и тест-архивами),
      `make run` (подсказка запуска MAME), `make clean`.
- [x] Заголовок DSS EXE (`ORG #4000`, код `#4200`, стек `#7FFE`), точка входа.
- [x] `include/{exe_header,dss_equ,bios_equ,ports}.inc` (взяты из gifview, проверенные).
- [x] Программа печатает баннер (`Dss.PChars`) и выходит (`Dss.Exit`).
**Проверка:** EXE-заголовок верен (магия/версия, load+entry #4200, стек #7FFE,
код начинается с `LD SP,#7FFE`); образ дискеты собирается с тест-архивами.

## Этап 1 — CLI, разбор заголовков, режим списка (экран + файл) + маски  [x]
**Цель:** полноценный командный интерфейс и оглавление архива (как `lha l`).
- [x] Чтение командной строки DSS (через `IX`): парсер токенов `GetCmdParam`.
- [x] Разбор флагов `-`/`/` `l/o/s/x`; позиционные `<archive>`,
      `[<out_dir>|<list_file>]`, `[<mask>]`. Печать `Usage` при ошибке.
- [x] Открытие архива (`Dss.Open`), чтение (`Dss.Read`), `Dss.Close`,
      перемотка к след. записи 32-битным `Move_FP` FromCurrent (HL:IX).
- [x] Парсер заголовков Level 0 и Level 1 (метод, размеры, имя). Единая
      формула след. записи `(2+headerSize)+packed` (проверена на всех `test/`).
- [x] Движок масок: `*`/`?`, регистронезависимо, `/`≡`\`, завершающий `\`→`\*`.
- [x] Режим `-l`: вывод на экран с пейджером (22 строки, `Esc` = стоп);
      при заданном `<list_file>` — запись листинга в файл (`Dss.Create`/`Dss.Write`).
- [x] Формат строки: метод, упак. размер, исходный размер, имя (32-бит десятич.).
- [ ] (отложено в этап 6: дата/время, явный разбор Extended Headers, `-lhd-`.)
**Проверка:** парсинг заголовков сверен с эталоном на всех `test/` (см. ниже
«как тестировать»). Запуск на эмуляторе/железе — на стороне пользователя.

## Этап 2 — Метод -lh0- (хранение): запись, CRC16, каталог вывода, фильтры
**Цель:** полноценная распаковка без декомпрессии и вся инфраструктура вывода.
- [ ] Выделение буферов через `DssGetMem` + маппинг физ. страниц (`RST #08`, EMM FN4) в WIN2/WIN3.
- [ ] Каталог вывода: по умолчанию текущий (`GetCurDir`); либо позиционный
      `<out_dir>` → абсолютный путь + завершающий `\` (создание подкаталогов).
- [ ] Создание выходного файла (`DssCreate`), потоковая запись (`DssWrite`).
- [ ] Применение маски к извлекаемым записям; флаги `-o` (перезапись),
      `-s` (пропуск существующих), `-x` (срез префикса каталога из маски,
      `DeriveStripPrefix`/`ApplyStripPrefix`).
- [ ] Копирование `-lh0-` вход→выход через буферы.
- [ ] Реализация CRC16 (полином 0xA001, init 0), сверка с заголовком.
- [ ] Коды возврата (0/1/3/7/13/23 + проброс DSS) и сообщения об ошибках;
      сохранять первую ненулевую ошибку; опрос прерывания в петле распаковки.
**Проверка:** распаковка `-lh0-`-записи (или синтетического архива) в текущий и
в заданный каталог; маски и `-o`/`-s`/`-x` работают; CRC16 совпадает, файл
байт-в-байт равен оригиналу; коды возврата соответствуют ситуациям.

## Этап 3 — Декодер -lh5- (AR002), эталонная реализация в DRAM
**Цель:** корректная распаковка самого распространённого метода (без SRAM-оптимизации).
- [ ] Чтение битового потока MSB-first.
- [ ] Разбор блока: число символов, 3 таблицы Хаффмана (длины кодов → таблица P →
      главная таблица литералов/длин → таблица расстояний).
- [ ] Построение и обход таблиц декодирования Хаффмана.
- [ ] LZSS: литерал → байт; пара (длина 3..256, расстояние) → копирование из
      скользящего окна 8 KB; корректная обработка перекрытий и переноса окна.
- [ ] Стрим: дочитывание входных блоков и сброс выходных по мере заполнения.
**Проверка:** `clock.lzh`, `wios110r.lzh` распаковываются, CRC16 совпадает,
сравнение с эталоном (`lha`/`7z`) байт-в-байт. **Это функциональный MVP-минимум.**

## Этап 4 — Декодер -lh1- (LZHUF), эталонная реализация в DRAM
**Цель:** второй распространённый метод (присутствует в тестах).
- [ ] Адаптивный Хаффман: построение/перестройка дерева, инкремент частот, ребаланс.
- [ ] Декодирование расстояния (6-битный префикс + явные младшие биты), окно 4 KB,
      длина 3..60.
- [ ] Интеграция в общий конвейер вывода и CRC16.
**Проверка:** `CPM431.lzh`, `GS19.LZH`, `WINPROFI.LZH` распаковываются, CRC16
совпадает, сравнение с эталоном. **MVP закрыт: -lh0-, -lh5-, -lh1-.**

## Этап 5 — Перенос горячего кода в SRAM CACHE (ускорение)
**Цель:** максимальная скорость без изменения функциональности.
- [ ] `EnterCacheWindow` / `RestoreSystemWindow`, инициализация SRAM-блока.
- [ ] Вынести в `src/cache_code.asm` под `DISP`: петлю декодера -lh5-/-lh1-,
      чтение битов, обход Хаффмана, копирование совпадений, обновление CRC16.
- [ ] Таблицы Хаффмана/дерево и таблица CRC — в SRAM.
- [ ] Границы DSS-ввода-вывода: `Load…Cache`/`Write…Cache` (Restore→DSS→Enter).
- [ ] `ASSERT` на раскладку SRAM (#0000-#3FFF).
**Проверка:** результаты идентичны этапам 3–4 (CRC/байты), измеримое ускорение
(сравнить время на одних и тех же файлах до/после).

## Этап 6 — Прочие AR002-методы и удобство (после MVP)
**Цель:** расширение покрытия и UX.
- [ ] -lh4- (окно 4 KB), -lh6- (32 KB), -lh7- (64 KB) — переиспользовать кодек
      AR002, добавить многостраничное кольцевое окно через WIN2/WIN3.
- [ ] Опционально -lzs-/-lz5- (LArc LZSS).
- [ ] Распаковка нескольких записей за запуск; создание подкаталогов из `-lhd-`.
- [ ] Уровни заголовков 2 и 3.
- [ ] (CLI базово готов на этапах 1–2: `-l`/`-o`/`-s`/`-x`, маски, out_dir,
      list_file — здесь только доработки удобства, если потребуются.)
**Проверка:** многозаписевые и многокаталожные архивы распаковываются полностью
с верными CRC16.

## Этап 7 — Стабилизация и релиз
- [ ] Регрессионный прогон по всем `test/` (CRC16 + сравнение с эталоном).
- [ ] Обработка повреждённых/обрезанных архивов без зависаний.
- [ ] Документация использования, коды возврата, README.
- [ ] Финальная сборка `UNLHA.EXE` + образ дискеты.

### Порядок и зависимости
0 → 1 → 2 → 3 → 4 — это MVP (распаковка -lh0-/-lh5-/-lh1- корректно).
5 — оптимизация поверх готового MVP (поведение не меняет).
6, 7 — расширения и релиз.

### Заметки по проверке производительности (этап 5)
Замерять одинаково до/после: один и тот же набор файлов, turbo-режим,
фиксировать время распаковки (например, по таймеру/счётчику кадров) и
сравнивать DRAM-версию (этапы 3–4) с SRAM-версией.
