# UNLHA — распаковщик LZH/LHA для Sprinter DSS

Консольная утилита для Sprinter DSS, распаковывающая архивы **LZH/LHA**. Написана
на ассемблере Z80 (sjasmplus) с упором на скорость: «горячий» код декодера и таблицы
исполняются из **SRAM-кэша** (окно 0, 0 wait-состояний в turbo 21 МГц).

## Возможности

- Методы сжатия: **`-lh0-`/`-lz4-`** (хранение), **`-lh5-`** (AR002, окно 8 КБ),
  **`-lh1-`** (LZHUF, окно 4 КБ).
- Заголовки **Level 0 и 1** (в т.ч. Amiga-архивы: каталог из extended header,
  длинные имена приводятся к DOS 8.3).
- Контроль целостности — **CRC16/ARC** (полином `0xA001`) по распакованным данным.
- Подкаталоги создаются автоматически (путь из extended header type 0x02).
- Выборочная распаковка по маскам; запрос на перезапись существующих файлов.

## Синтаксис

```
unlha.exe <archive.lzh> [<out_dir>] [<mask>]            ; распаковать
unlha.exe -l <archive.lzh> [<list_file>] [<mask>]      ; список (экран/файл)
unlha.exe -o <archive.lzh> [<out_dir>] [<mask>]        ; перезаписывать
unlha.exe -s <archive.lzh> [<out_dir>] [<mask>]        ; пропускать существующие
unlha.exe -x <archive.lzh> [<out_dir>] <mask>          ; срезать префикс каталога маски
```

Флаги (префикс `-` или `/`, односимвольные): `-l` список, `-o` перезапись,
`-s` пропуск, `-x` срез префикса. Флаги комбинируются с позиционными аргументами.

### Каталог распаковки
По умолчанию — текущий. Можно задать позиционным `<out_dir>` (приводится к
абсолютному пути). Если второй аргумент содержит маску (`*`/`?`) — он трактуется
как фильтр, вывод идёт в текущий каталог.

### Маски
`*` — любая последовательность, `?` — один символ; сравнение регистронезависимое;
`/` и `\` эквивалентны; маска, оканчивающаяся на `\`/`/`, означает «всё в каталоге».

### Список (`-l`)
Без `<list_file>` печатается на экран с постраничной паузой (Esc — стоп). С именем
файла — пишется в текстовый файл. Формат строки: метод, упакованный и исходный
размеры (по правому краю), дата/время (`YYYY-MM-DD HH:MM`), имя.

### Перезапись
Без `-o`/`-s` при существующем файле спрашивает: `Overwrite? y/n, a=all, s=skip`
(`a` — перезаписывать все дальше, `s` — пропускать все, Esc/Ctrl+C — прервать).

### Прерывание
`Esc` или `Ctrl+C` прерывают список и распаковку (печать `Aborted`, код возврата 1).

## Коды возврата

| Код | Значение |
|-----|----------|
| 0   | успех |
| 1   | неверная командная строка либо прерывание (Esc/Ctrl+C) |
| 3   | архив не найден |
| 7   | выходной файл существует и не заменён |
| 13  | повреждён/неизвестен формат LZH |
| 23  | несовпадение CRC16 |
| —   | прочее: проброс кода ошибки DSS (сохраняется первая ненулевая) |

## Сборка

Требуются `sjasmplus` и `mtools`.

```
make exe     # build/UNLHA.EXE (DSS EXE: ORG #4200, стек #7FFE)
make img     # build/unlha.img — образ дискеты с EXE и тест-архивами
make clean
```

## Ограничения

- Не поддержаны методы `-lh4-`/`-lh6-`/`-lh7-` (окна 4/32/64 КБ), `-lzs-`/`-lz5-`
  (LArc), а также заголовки **Level 2/3** — в тестовом наборе таких архивов нет,
  проверить реализацию нечем.
- Имена приводятся к DOS 8.3 (база ≤8, расширение ≤3); недопустимые символы → `_`.
- Повреждённые/обрезанные архивы не вызывают зависания (выдают CRC ERROR), но
  специальной диагностики формата нет.

## Тестовые архивы

`test/` содержит `-lh5-` (clock, wios110r, Amiga-игры `*.lha`),
`-lh1-` (CPM431, GS19, WINPROFI) и `-lh0-` записи. Критерий корректности —
совпадение CRC16 с заголовком и верный размер.
